### PR #345 — Critical rendering fix: remove `<details>` blocks
- **Root cause**: kramdown HTML-escapes `</summary>` and `</details>` closing tags inside `<details>` blocks containing markdown code fences with blank lines
- **Effect**: Unclosed `<summary>` elements cascaded, swallowing all page content after Section 1
- **Fix**: Removed all `<details class="mermaid-source">` blocks from all 4 pages
- These blocks were already hidden by CSS (`display: none !important` in publication.html)
- Mermaid source preserved in raw markdown files
- Merged to main

## Lessons
- kramdown does not reliably handle nested HTML with markdown code fences inside `<details>` blocks
- User had warned about collapsible `<details>` being problematic — should have listened first (Pitfall #20)
- Pre-rendered PNG diagrams with `<picture>` elements work correctly without the `<details>` source blocks

## Decisions
- Mermaid source blocks removed from web pages but preserved in publication source files
- No need for collapsible source display — CSS was hiding them anyway
