## Remember

- **Add creation date, modification date, and version** to all documentation and publications when created and updated. Every doc/publication should track: creation date, last modification date, and version number.
- **Conformity methodology for doc/pub/webcard creation** — When creating or updating publications, follow the established conformity pattern: (1) three-tier structure (source → summary → complete), (2) bilingual EN/FR mirrors, (3) update all references (indexes, profiles, CLAUDE.md), (4) webcards follow the same content-inspired design philosophy — read the publication first, design the webcard to visualize its core concept. Static generation for most cards, dynamic generation (parse structured data from source) for live-data publications like #4a. The webcard, doc, and pub creation all share the same principle: content drives design, not templates.
- **Always use fenced code blocks for structured output** — Any command output containing URLs, command tables, inventories, or data the user may want to copy must be wrapped in triple-backtick code blocks. Applies to: `weblinks`, `weblinks --admin`, `help`, `pub list`, `harvest --list`, `harvest --procedure`, `docs check`, and any similar structured output.
- **One code block per URL** — Claude Code's copy button copies the entire code block. URLs must each be in their own fenced code block for reliable copy. Labels, headers, severity icons go outside as plain markdown. Never put multiple URLs in one code block.
