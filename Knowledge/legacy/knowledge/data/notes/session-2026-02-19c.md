## Methodology Discovery — Two-Merge Bootstrap Lifecycle

Bootstrap over-sync analysis revealed that ~60% of the v22 bootstrap content duplicates core (Branch Protocol, Save Protocol, Session Lifecycle, Part 1 commands). These should be read from core on wakeup, not hardcoded in satellite CLAUDE.md.

**The proper satellite lifecycle is three stages, two merges:**

```
Stage 1: Bootstrap → PR → merge to main     ← install v22 pointer + initial sync
Stage 2: Wakeup + Normalize → PR → merge    ← trim bloat, apply thin-wrapper pattern
Stage 3: harvest --healthcheck               ← scan clean satellite from core
```

**Satellite thin-wrapper principle**: A satellite CLAUDE.md should be ~30 lines (bootstrap pointer + project overview + project-specific commands). Universal content is inherited from core on wakeup, never duplicated. Over-sync during initial bootstrap is expected — `normalize` is the cure.
