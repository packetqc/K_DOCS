## Abstract

Software engineering with AI assistants suffers from a fundamental limitation: **statelessness**. Every new session starts from zero. When work spans multiple sessions across multiple projects, the accumulated intelligence is scattered and unreachable.

**Knowledge** (`packetqc/knowledge`) solves this by creating a self-evolving intelligence layer for AI-assisted engineering. **By design**, it only operates on repositories that the user owns and that Claude Code has been granted access to — no external or third-party repos are ever accessed.

| Feature | Description |
|---------|-------------|
| **Session persistence** | 30-second context recovery vs 15 minutes manual |
| **Portable bootstrap** | Any project inherits methodology, commands, patterns on wakeup |
| **Bidirectional flow** | Satellites evolve independently; harvest pulls discoveries back |
| **Version-aware** | 47 versions tracked; drift detected and remediated automatically |
| **Self-aware** | Living dashboard shows network health in real time |
| **Human Person Machine Bridge** | GitHub Projects v2 as Jira-style tracking; GitHub Pages as Confluence-style documentation — $0/month replaces $14.20/user/month |
| **13 core qualities** | *Self-sufficient, autonomous, concordant, concise, interactive, evolutionary, distributed, persistent, recursive, secure, resilient, structured, integrated* |
| **`#` call alias** | Single-character scoped knowledge input with implicit main project and multi-satellite convergence |
| **Time compilation** | Structured delivery timing: Active Session vs Calendar Elapsed vs Enterprise equivalent, with CSS pie charts. Real data from Knowledge (git history) and the human user (domain expertise) — modern time tracking without SaaS dashboards |

> *"Even the provider Claude Code AI uses Atlassian Confluence. For managing projects distributed across multiple entities and for distributed web publishing, I use Knowledge, which is built on a Claude Code AI plan paid monthly for conception, then GitHub lives with Knowledge."*
> — Martin Paquet, architect of Knowledge

This is the **master publication** — built by harvesting its own 15 children. The [13 core qualities]({{ '/publications/knowledge-system/full/#core-qualities' | relative_url }}) are the system's DNA — each discovered through practice, each reinforcing the others.

## What the System Contains

Knowledge is organized in layers, from stable core methodology to ephemeral session memory. Each layer serves a different purpose in the system's lifecycle.

| Layer | Content | Stability |
|-------|---------|-----------|
| **Core** (CLAUDE.md) | Identity, methodology, v1–v47 evolution, 13 qualities, `#` call alias | Stable |
| **Proven** (patterns/, lessons/) | 4 patterns, 18 pitfalls | Validated |
| **Harvested** (minds/) | 24 candidates from 5 satellites | Evolving |
| **Publications** | 15 technical publications | Versioned |
| **Projects** (projects/) | 9 projects (P0–P9) with hierarchical indexing | Structured |
| **Tooling** (live/, scripts/) | Capture engine, beacon, scanner, webcard generator | Portable |

The layers work together: core provides identity, proven knowledge provides patterns, harvested knowledge provides fresh insights, and session memory provides continuity.
